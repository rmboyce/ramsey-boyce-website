Overview

This program generates crossword puzzles in three steps: grid generation, word filling, and cluing. The puzzle is served dynamically using Flask.

Generating the Puzzles

Grid generation: the code starts with an empty grid, adds rows and columns extending inwards, and randomly fills the center. Rotational symmetry is enforced by mirroring the first half.

Word filling: a tree search is used to match words to slots. Because brute-force search is slow, optimizations in both search heuristics and execution speed were implemented.

Search heuristics:

  • Prune search branches immediately when a slot has zero valid words.
  • Fill slots with the fewest possible word options first.
  • Prioritize words that leave the most options open for crossing slots.

Code efficiency:

  • Use a thread pool to parallelize word selection without thread-spawning overhead.
  • Store word lists as SIMD vectors for fast pattern matching.
  • Minimize data copying.

These optimizations reduced fill time to 0.1 seconds—a 1000x speedup over previous iterations.

Cluing: clues are matched from archives of historical crosswords rather than generated algorithmically.

Playing on my Website

Flask serves a new puzzle daily, and a JavaScript player from Crossword Nexus renders the grid in-browser.
